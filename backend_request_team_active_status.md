백엔드 팀 전달 사항: 팀 생성 시 '활성 상태' 반영 요청

날짜: 2025년 12월 10일

안녕하세요 백엔드 팀,

프론트엔드에서 팀 생성 시 '비활성' 상태로 설정해도 팀이 계속 '활성' 상태로 생성되는 문제가 발생하고 있습니다.

**프론트엔드 변경 사항 요약:**
*   `CreateTeamRequest` DTO에 `active?: boolean` 필드를 추가하여 팀 생성 요청 시 활성 상태를 명시적으로 보낼 수 있도록 업데이트했습니다.
*   프론트엔드에서는 이제 사용자가 '비활성'을 선택할 경우 `active: false`를 포함하여 `/api/teams` POST 요청을 보내고 있습니다. (예: `{"name": "새 팀", "description": "설명", "active": false}`)
*   '활성'을 선택하거나 선택하지 않을 경우 (기본값) `active: true`를 보내거나 `active` 필드를 생략합니다.

**백엔드 요청 사항:**
*   `/api/teams` (POST) 엔드포인트에서 팀 생성 시 요청 바디의 `active` 필드를 존중하도록 처리를 확인해 주시기 바랍니다.
*   `active: false`로 요청이 들어올 경우 해당 팀은 비활성 상태로 생성되어야 합니다.
*   `active` 필드가 요청 바디에 포함되지 않은 경우 (현재 기본 동작처럼) 활성 상태(`true`)로 생성되어도 좋습니다.

이 문제 해결을 통해 프론트엔드에서 팀 생성 시 활성 상태를 정확하게 제어할 수 있도록 지원 부탁드립니다.

감사합니다.
