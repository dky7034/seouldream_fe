### 제목: [백엔드] 통계 API 정책 변경 안내

안녕하세요, 백엔드 팀입니다.

통계 관련 API의 정책이 최종 확정되어 관련 내용을 안내해 드립니다.

#### 1. 주요 변경 내용 요약

*   **정책 확정:** 셀장(CELL_LEADER)은 **오직 현재 연도(current year)의 통계 데이터만 조회**할 수 있는 정책이 확정 및 반영되었습니다.
*   **기능 개선:** 교역자(EXECUTIVE)는 연도 필터 API(`available-years`) 사용 시 특정 셀을 지정하여 조회하는 기능이 추가되었습니다.

#### 2. API 변경 상세 (`GET /api/statistics/available-years`)

연도 필터 목록을 조회하는 API의 사용법은 이전과 동일하게 유지되며, 교역자를 위한 기능이 추가되었습니다.

*   **교역자 (`EXECUTIVE`)일 경우:**
    *   **전체 연도 목록**이 필요할 때: 파라미터 없이 호출합니다.
        *   `GET /api/statistics/available-years`
    *   **특정 셀의 연도 목록**이 필요할 때: `cellId` 쿼리 파라미터를 추가하여 호출합니다.
        *   `GET /api/statistics/available-years?cellId=123`

*   **셀장 (`CELL_LEADER`)일 경우:**
    *   `cellId` 파라미터와 관계없이, 항상 **자신이 속한 셀의 데이터가 있는 모든 연도 목록**을 반환합니다.

#### 3. 셀장 데이터 조회 정책 안내

*   **매우 중요:** 셀장이 통계 데이터(예: `/api/statistics/overall-attendance`)를 조회할 때, **반드시 현재 연도의 데이터만 조회 가능합니다.**
*   `available-years` API 호출 결과에 과거 연도가 포함되어 있더라도, 해당 과거 연도로 실제 데이터 조회를 요청하면 백엔드에서 **"셀장은 현재 연도의 데이터만 조회할 수 있습니다."** 라는 오류 메시지와 함께 요청이 거부됩니다.

#### 4. 프론트엔드 요청 사항

*   셀장(CELL_LEADER)이 로그인했을 경우, 데이터 조회 기능(통계, 출석 추이 등)의 **연도 필터를 현재 연도로 비활성화(disabled)하거나, 과거 연도를 선택할 수 없도록 UI/UX적으로 처리**해 주시길 권장합니다. 이는 사용자가 заведомо 실패할 요청을 보내는 것을 방지하기 위함입니다.
*   교역자(EXECUTIVE) 화면의 경우, `available-years` API의 신규 기능을 활용하여 특정 셀 필터링 시 연도 목록을 동적으로 업데이트하는 로직을 적용할 수 있습니다.

감사합니다.
